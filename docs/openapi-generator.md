# OpenAPI Generator

`protoc-gen-connect2openapi` generates one OpenAPI v3.1 JSON document for the
REST routes generated by `protoc-gen-connect2rest`.

The generator currently delegates proto-to-OpenAPI schema/comment generation to
grpc-gateway's `protoc-gen-openapiv3`, then patches the result for
connect2axum-specific behavior:

- merged output when grpc-gateway emits one document per proto file;
- config-driven `info`, `servers`, security schemes, root security, and global
  header parameters;
- generated REST DTO body schemas that grpc-gateway cannot see directly;
- NDJSON content types for streaming REST endpoints;
- package-prefix suppression for schema components and references;
- final OpenAPI v3.1 parsing through `oas3`.

This keeps ProtoJSON-compatible schema behavior aligned with grpc-gateway while
leaving the backend isolated enough to replace later if the AsyncAPI work proves
we need a native schema core.

## Buf Usage

```yaml
plugins:
  - local: protoc-gen-connect2openapi
    out: src/generated/openapi
    strategy: all
    opt:
      - config=connect2openapi.yaml
```

## Plugin Options

Options are comma-separated `name=value` pairs.

| Option | Default | Purpose |
| --- | --- | --- |
| `output` | `openapi.json` | Generated output file name. |
| `config` | none | YAML config file for metadata, servers, security, headers, and streaming content type. |
| `openapiv3_bin` | auto-detect | Path to grpc-gateway's `protoc-gen-openapiv3` binary. |
| `openapiv3_opt` | none | Extra option forwarded to `protoc-gen-openapiv3`; may be repeated. |
| `streaming_content_type` | `application/x-ndjson` | Fallback REST streaming content type. |
| `suppress_pkg_prefix` | `true` | Omit protobuf package prefixes from schema component names and `$ref`s. |

`disable_default_errors=true` is forwarded to `protoc-gen-openapiv3` unless an
explicit `disable_default_errors=...` option is supplied via `openapiv3_opt`.
When `suppress_pkg_prefix=true`, two schema names that shorten to the same
component name are a fatal generation error. Set `suppress_pkg_prefix=false` to
keep fully qualified protobuf names.

Binary lookup order for `protoc-gen-openapiv3`:

1. `openapiv3_bin=...`
2. `CONNECT2AXUM_OPENAPIV3_BIN`
3. `PATH`
4. `$HOME/go/bin/protoc-gen-openapiv3`

## Config File

The config file is optional YAML using camelCase keys.

```yaml
info:
  title: Simple API
  version: 0.1.0
  description: Simple connect2axum example API.
  termsOfService: https://example.com/terms

servers:
  - url: http://127.0.0.1:8000/v1
    description: Local REST server

securitySchemes:
  BearerAuth:
    type: http
    scheme: bearer
    bearerFormat: JWT

security:
  - BearerAuth: []

headers:
  - name: X-Request-Id
    required: false
    description: Optional request correlation ID.
    schema:
      type: string

streamingContentType: application/x-ndjson
```
