# connect2axum WebSocket Streaming Example

This example exercises optional JSON WebSocket routes generated by
`protoc-gen-connect2ws` beside the REST routes generated by
`protoc-gen-connect2rest`. Generated code is checked in so reviewers can inspect
the Buffa, Connect Rust, REST, and WebSocket output.

Required local plugins:

```sh
cargo install --locked protoc-gen-buffa protoc-gen-buffa-packaging connectrpc-codegen
cargo install --path crates/connect2axum-codegen
```

From this example directory:

```sh
buf lint
buf generate
```

From the workspace root:

```sh
cargo test -p ws-streaming --all-features
cargo run -p ws-streaming
```

WebSocket server-streaming endpoint with `websocat`:

```sh
printf '%s\n' '{"firstName":"Jane","lastName":"Doe"}' |
websocat -t ws://127.0.0.1:8002/v1/hello/expand/ws
```

WebSocket bidirectional-streaming endpoint with `websocat`:

```sh
printf '%s\n%s\n' \
  '{"firstName":"Jane","lastName":"Doe"}' \
  '{"firstName":"Ada","lastName":"Lovelace"}' |
websocat -t ws://127.0.0.1:8002/v1/hello/chat/ws
```

WebSocket client-streaming endpoint. The blank final line sends an empty text
frame, which ends the request stream while keeping the socket open for the
unary response:

```sh
printf '%s\n%s\n\n' \
  '{"firstName":"Jane","lastName":"Doe"}' \
  '{"firstName":"Ada","lastName":"Lovelace"}' |
websocat -t ws://127.0.0.1:8002/v1/hello/collect/ws
```

REST server-streaming endpoint:

```sh
curl -N -i \
  -X POST 'http://127.0.0.1:8002/v1/hello/expand' \
  -H 'content-type: application/json' \
  --data '{"firstName":"Jane","lastName":"Doe"}'
```

Connect protocol endpoint with `curl` uses Connect's streaming envelope, so the
response is binary-framed:

```sh
python3 - <<'PY' |
import json
import struct
import sys

msg = json.dumps({"firstName": "Jane", "lastName": "Doe"}, separators=(",", ":")).encode()
sys.stdout.buffer.write(b"\0" + struct.pack(">I", len(msg)) + msg)
PY
curl -N -s \
  -X POST 'http://127.0.0.1:8002/streaming.v1.GreeterService/Expand' \
  -H 'content-type: application/connect+json' \
  -H 'connect-protocol-version: 1' \
  --data-binary @- | xxd -g 1
```

gRPC endpoint with `grpcurl`:

```sh
grpcurl \
  -plaintext \
  -import-path examples/ws-streaming/proto \
  -proto streaming/v1/ws_streaming.proto \
  -d '{"firstName":"Jane","lastName":"Doe"}' \
  127.0.0.1:8002 \
  streaming.v1.GreeterService/Expand
```
